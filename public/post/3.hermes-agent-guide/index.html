前言

我是在 Arch Linux (CachyOS) + niri (Wayland Compositor) 环境下日常使用的开发人员。之前试过一些 AI 编程助手，但总感觉停留在"帮你写代码"的层面。直到最近深度体验了 Hermes Agent，才发现终端 AI 助手能做的远不止这些。

几个真实场景

1. 排查快捷键失灵的 Bug

某天我的 Win+Z 快捷键突然没反应了。以前遇到这种问题，我得翻日志、查进程、Google 一圈。

Hermes 直接帮我：

  • 定位到快捷键绑定在 niri/config.d/70-binds.kdl，调用了 inir overview toggle
  • 手动执行发现 IPC 超时，确认是 Quickshell 的 IPC Handler 死锁
  • 找到解决方案：inir restart 即可，不用重启电脑
  • 自动把排查过程整理成笔记存到 Obsidian

2. 整理杂乱无章的笔记库

我的 Obsidian 笔记库 40 多个目录，命名混乱、分类重叠。我说了一句"太乱了"，Hermes ：

  • 分析了所有文件和目录结构
  • 给出了重组方案
  • 一键完成迁移：创建新目录、移动文件、清理空目录、git commit 一条龙
  • 过程中我要求调整（“有些笔记应该放问题排查”、“一万个坑拆开”），都能准确理解并执行

3. 检查幽灵进程

我总觉得系统里有"莫名其妙的 Python 进程"，Hermes 帮我逐个排查：

  • 定位到每一个 Python 进程的路径和用途
  • 发现 geoclue-demo-agent 是无用的定位演示程序
  • 查清依赖链，解释为什么会被装上、删了会不会影响桌面
  • 分析排序显式安装的包，一眼看出哪些占空间但没用

4. 随手存笔记

排查完 Bug 后说"帮我存到 Obsidian"，自动根据内容选择合适的目录放好，还记住了路径，下次说"存笔记"就自动处理。

为什么我觉得它好用

  • 有上下文意识：不是一问一答的机器人，能理解"刚才我们聊到的那件事"
  • 有判断力：我导入的 skill 它不会盲目执行，会说明能力边界
  • 会帮你记住：保存常用路径、偏好设置，不用每次都重复说明
  • 真的能操作：移动文件、编辑配置、重启服务，不是只给建议
  • 出错会自己修正：发现 skill 过时了会直接更新，不用等我指出来

一些槽点（实话实说）

  • 跑在终端里，没有 GUI，有门槛
  • 偶尔上下文窗口满了需要 /new 重开
  • 对 Qt/QML 相关的底层 bug 也只能定位到表象，修还是要等上游

总结

如果你已经是 Linux 终端用户，Hermes Agent 能把你"想但懒得做"的事情真正做掉——排查问题、整理文件、写笔记、审计系统。它不是帮你写代码的，是帮你管理电脑的。

安装方式参考官方文档：https://hermes-agent.nousresearch.com/docs
